# Patient dashboard mobile panels

The patient dashboard is a webapp but must feel complete on mobile.

## Rule

Mobile users should not lose functions that are only available in the desktop side panel.

The phone/app screen must include:

- today's exercise path;
- completion form;
- pain score entry;
- AI Movement Check link when enabled;
- progress summary;
- messages from the physiotherapist;
- bottom navigation to today/progress/messages/logout.

## Footer rule

The public marketing footer is hidden after the patient dashboard so the patient experience feels like a real app screen, not a public website page.

## Bottom navigation

The bottom nav should point to real sections inside the mobile app screen:

- `#path` for today's plan;
- `#progress` for progress summary;
- `#messages` for physiotherapist messages.
