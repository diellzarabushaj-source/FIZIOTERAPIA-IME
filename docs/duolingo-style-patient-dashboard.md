# Duolingo-style patient dashboard rules

Patient Dashboard should feel like a simple mobile learning path, not a complicated medical software screen.

## UI principles

- Mobile-first.
- One clear next action.
- Large touch targets.
- Short Albanian copy.
- Big friendly cards.
- A vertical path of exercises.
- Bottom navigation always available on mobile.
- Safety rule visible without searching.

## Keep all functions, but simplify the presentation

The dashboard still supports:

- patient code login;
- assigned exercises;
- completion logging;
- pain score after exercise;
- optional comment to physiotherapist;
- AI Movement Check when enabled;
- messages from physiotherapist;
- progress, pain and AI trends.

But the patient should mainly see:

1. `Fillo ushtrimin`
2. `Çka ke sot`
3. `E kryva`
4. `Dhimbja pas ushtrimit`
5. `AI opsionale`

## Safety

Pain 7/10 or higher means stop and contact the physiotherapist.

AI is optional movement feedback only. It does not diagnose, prescribe, stop, or change therapy.
